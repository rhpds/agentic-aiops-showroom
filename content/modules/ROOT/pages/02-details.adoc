=== Setup validation

Run these commands from the Showroom terminal to verify your environment:

[source,bash]
----
# Verify OpenShift CLI access
oc whoami

# Verify your namespace is active
oc project {aap2_organization}

# Test Red Hat MaaS API connectivity
curl -s {litellm_api_base_url}/models \
  -H "Authorization: Bearer {litellm_virtual_key}" \
  | python3 -m json.tool | grep '"id"'

# Verify Gitea API access
curl -s {gitea_url}/api/v1/user \
  -u {gitea_username}:{gitea_password} \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['login'])"
----

== Troubleshooting guide

=== Common setup issues

*Problem*: `Error from server (Forbidden): namespaces is forbidden`
→ *Solution*: Re-login as your workshop user: `oc login {openshift_api_url} -u {aap2_username} -p {aap2_password}`

*Problem*: `401 Unauthorized` from MaaS API
→ *Solution*: Confirm you are using the correct virtual key for `{litellm_key_alias}`.
Keys are valid for {litellm_key_duration} from provisioning. If expired, contact your facilitator.

*Problem*: Cannot reach Gitea URL
→ *Solution*: Open {gitea_url} directly in your browser first.
If unreachable, the ingress domain `{openshift_cluster_ingress_domain}` may be temporarily unavailable — notify your facilitator.

*Problem*: RocketChat or Kira not loading
→ *Solution*: Both are routed through `{openshift_cluster_ingress_domain}`. Hard-refresh your browser (Ctrl+Shift+R / Cmd+Shift+R) and try again. If the issue persists, notify your facilitator.

*Problem*: RHEL VM not responding to Ansible
→ *Solution*: Verify the VM hostname is reachable: `ping {rhel_vm_hostname}` from your terminal.
The VM is accessible only from within the cluster network via the Showroom terminal.
